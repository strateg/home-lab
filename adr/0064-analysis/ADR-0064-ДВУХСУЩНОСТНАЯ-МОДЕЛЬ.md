## ✅ ЧТО СДЕЛАНО

ADR 0064 полностью переработан под вашу концепцию:

### Было (старая версия)
- Один класс: `os` (с подклассами `os.firmware` и `os.installable`)
- Путаница: firmware как "OS", а не как отдельная сущность

### Стало (новая версия)
- **Два независимых класса:**
  1. `class: firmware` → программная основа устройства
  2. `class: os` → операционная система поверх firmware

### Ключевые Различия

| Аспект | Старая модель | Новая модель |
|--------|--------------|--------------|
| **Firmware** | Не моделировался | Отдельный класс |
| **OS** | Один класс с подклассами | Отдельный класс |
| **Семантика** | `class: os.firmware` | `class: firmware` + `class: os` |
| **Object naming** | `routeros-7-firmware` | `mikrotik-routeros7-firmware` (firmware) + `routeros-7` (os) |
| **PDU** | "OS-less device" | Firmware-only device |
| **PC BIOS** | Не моделировался | `generic-uefi-x86-firmware` |
| **Capabilities** | Только `cap.os.*` | `cap.firmware.*` + `cap.os.*` |

## 🎯 ДВУХСЛОЙНАЯ МОДЕЛЬ

```
Device
  ├─ bindings.firmware (required) ← Программная основа
  └─ bindings.os (conditional)    ← Операционная система
```

### Firmware (низкий уровень)
- Всегда есть у активных устройств
- Vendor/hardware-specific
- Примеры: BIOS, UEFI, RouterOS firmware, PDU mgmt firmware

### OS (высокий уровень)
- Может быть или нет (по политике класса)
- Embedded (часть firmware) или installable (независимый)
- Примеры: RouterOS 7, Debian 12, macOS 14, Windows 11

## 📊 CAPABILITIES ОТ ДВУХ СЛОЕВ

```yaml
# MikroTik Router
bindings:
  firmware: obj.firmware.mikrotik-routeros7
  os: obj.os.routeros-7

capabilities:
  from_firmware:
    - cap.firmware.mikrotik
    - cap.firmware.routeros
    - cap.firmware.arch.x86_64
  from_os:
    - cap.os.routeros
    - cap.os.routeros.7
    - cap.os.embedded
```

## 🔧 РЕАЛЬНЫЕ ПРИМЕРЫ

### PC (single-boot)
```yaml
firmware: generic-uefi-x86-firmware
os_primary: debian-12-generic (installable)
```

### PC (dual-boot Windows + Linux)
```yaml
firmware: generic-uefi-x86-firmware
os_primary: windows-11 (installable)
os_secondary: debian-12-generic (installable)
```

### PC (triple-boot)
```yaml
firmware: generic-uefi-x86-firmware
os_primary: windows-11
os_secondary: debian-12-generic
os_tertiary: freebsd-14
```

### MacBook (NO multi-boot)
```yaml
firmware: apple-silicon-m2-firmware
os_primary: macos-14 (embedded, vendor-locked)
# os_secondary: NOT ALLOWED by vendor policy
```

### MikroTik Router
```yaml
firmware: mikrotik-routeros7-firmware
os_primary: routeros-7 (embedded, part of firmware)
```

### PDU
```yaml
firmware: apc-pdu-mgmt-firmware
# no OS (forbidden by policy)
```

## 💡 КЛЮЧЕВЫЕ ВЫВОДЫ

1. **Firmware ≠ OS**
   - Firmware — программная основа (всегда есть)
   - OS — операционная среда (может быть или нет)

2. **Две независимые сущности**
   - `class: firmware` → `object: mikrotik-routeros7-firmware`
   - `class: os` → `object: routeros-7`

3. **Capabilities от обоих**
   - Firmware: `cap.firmware.*`
   - OS: `cap.os.*`
   - Services могут требовать любые

4. **Четкие политики**
   - PC: firmware required, OS installable_allowed
   - MacBook: firmware required, OS embedded_only
   - MikroTik: firmware required, OS embedded_only
   - PDU: firmware required, OS forbidden

5. **PDU не "без OS", а "только firmware"**
   - Firmware есть (управление)
   - OS нет (не нужна)
