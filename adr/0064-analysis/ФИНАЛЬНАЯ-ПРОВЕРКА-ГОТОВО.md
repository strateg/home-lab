## ✅ ПРИМЕНЕННЫЕ ИСПРАВЛЕНИЯ

### 1. Канонический формат именования

**Исправлено:** Обновлен устаревший формат ссылок на объекты в секции Decision

**Было:**
```yaml
- `class: firmware` -> `object: mikrotik-routeros7-firmware`
```

**Стало:**
```yaml
- `class.firmware` -> `obj.firmware.mikrotik-routeros7` -> `inst.firmware.routeros-7-1`
```

### 2. Добавлены Categories

**Добавлено:** Таксономические категории во все определения классов

**Firmware:**
```yaml
categories: [infrastructure, prerequisite, hardware-bound]
```

**OS:**
```yaml
categories: [infrastructure, prerequisite, runtime]
```

**Добавлена секция:** "Entity Categories" с объяснением назначения и стандартного набора категорий

### 3. Свойство виртуального firmware

**Добавлено:** `virtual: boolean` в firmware class для поддержки VM

```yaml
properties:
  virtual: boolean  # true для виртуального firmware (VM)
```

### 4. Поддержка VM

**Добавлено:** Полная поддержка виртуальных машин

- Объекты виртуального firmware (KVM OVMF, VMware EFI, Hyper-V)
- Примеры VM device instances
- Обновлена таблица Real-World Examples
- Примечания о LXC/Docker (отложены на будущие ADR)

### 5. Модель instance references

**Проверено:** Все примеры используют правильный формат ссылок на instances

```yaml
instance: inst.compute.vm-app-01
object_ref: obj.compute.kvm-vm

firmware_ref: inst.firmware.kvm-ovmf-prod
os_refs: [inst.os.debian-12-prod]
```
