# SWOT: ADR0076 + ADR0081 Through ADR0089-0091 Lens

**Date:** 2026-04-09  
**Scope:** Evaluate extraction/artifact-first architecture (ADR0076, ADR0081) against SOHO product contracts (ADR0089, ADR0090, ADR0091), then define hardening actions.

---

## 1) Strengths

- Clear strict-gate foundation exists: lock/integrity (`E782x`) and phase evidence tooling already operational.
- Artifact-first flow already has practical bootstrap + verify + compile mechanics.
- ADR0089-0091 contracts are now machine-readable (`product_profile_state`, `operator-readiness`, `support-bundle-manifest`).
- Split-rehearsal lane now produces machine-readable summary for extracted-mode verification.

## 2) Weaknesses

- Historical gap: split-rehearsal focused mostly on lock/compile success, but not explicitly on SOHO product contract parity.
- Cutover checklist lacked explicit “SOHO contract parity green” condition.
- 0081 plan had no explicit bridge item binding P4 rehearsal to ADR0089-0091 acceptance outputs.
- Risk of false-green cutover: extracted compile may pass while product handover/readiness contract silently regresses.

## 3) Opportunities

- Use split-rehearsal summary as single source of truth for cutover readiness and SOHO parity.
- Convert ADR0089-0091 from “project-only checks” to “cross-mode invariants” (monorepo and extracted).
- Reuse same evidence bundle for architecture review, release sign-off, and incident triage.

## 4) Threats

- Cutover confidence without product evidence parity can cause release with degraded operator UX/runbooks.
- Drift between plans and runtime checks can recreate manual “checklist theater”.
- Future plugin/discovery changes can break SOHO outputs in extracted mode without failing core strict gates.

---

## 5) Hardening Actions

### Implemented now

1. Added split-rehearsal SOHO contract checks into machine-readable summary:
   - mandatory handover files,
   - mandatory report files,
   - ADR0091 D3 evidence domain coverage in `operator-readiness`.
2. Integrated split-rehearsal into Cutover evidence bundle (`framework:cutover-evidence`).
3. Extended tests:
   - split-rehearsal utility coverage,
   - cutover evidence bundle coverage with split-rehearsal outputs.
4. Updated Cutover and 0081 plans to explicitly require/track SOHO contract parity.

### Next priority improvements

1. Add deterministic parity comparison between monorepo and extracted `operator-readiness.json` (status + domain set).
2. Add cutover Go-condition requiring no critical `E794x` in split-rehearsal outputs.
3. Add CI job publishing `build/diagnostics/cutover/split-rehearsal.json` as mandatory artifact.

---

## 6) Decision

ADR0076/0081 remain sound for ADR0089-0091 rollout, but only if split-rehearsal is treated as product-contract gate, not only framework gate.  
Bridge hardening is now partially implemented; remaining items should be promoted into CI cutover policy.
