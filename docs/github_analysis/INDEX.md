## 🔄 Как использовать эти документы

### Сценарий 1: Начало работы над рефакторингом

1. Прочитайте **VALIDATORS_QUICK_REFERENCE.md** — 5 минут
2. Откройте **VALIDATORS_REFACTORING_TRACKER.md** в текстовом редакторе — он будет вашим спутником
3. Выберите следующую фазу из tracker
4. Приступите к кодированию

### Сценарий 2: Создание PR

1. Используйте **PRE_PR_CHECKLIST.md** — пройдите все шаги
2. Запустите `scripts/create_validators_pr.cmd` — автоматически создаст PR
3. Обновите **VALIDATORS_REFACTORING_TRACKER.md** — отметьте фазу как выполненную
4. Напишите PR body на основе template из PRE_PR_CHECKLIST

### Сценарий 3: Code review

1. Прочитайте **SESSION_SUMMARY_2026_02_25.md** — поймите, что было сделано
2. Прочитайте соответствующие разделы **VALIDATORS_REFACTORING_TRACKER.md**
3. Проверьте тесты и CI (используя **PRE_PR_CHECKLIST** как шаблон)

### Сценарий 4: Понимание архитектуры

1. Прочитайте **PROJECT_ANALYSIS.md** — полный обзор проекта
2. Прочитайте **adr/0045-model-and-project-improvements.md** — решения и план
3. Посмотрите исходные файлы в `topology-tools/scripts/validators/`
