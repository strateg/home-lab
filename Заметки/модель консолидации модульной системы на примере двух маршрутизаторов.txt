Предлагаю довести до ума систему плагинов и модулей на примере двух рутеров Mikrotik Chateau LTE7 ax и Gl.iNet Slate AX1800. Ожидается что первым шагом мы создаём class module в котором содержится definitions и plugins для работы с классом router. Вторым шагом мы создаём 2 object module. Первый объектный модуль связан с Mikrotik Chareau и его спецификой, плагины умеют работать со специфичными полями. Второй объектный модуль связан с GL.iNet Slate AX1800 и соответственно его спецификой, плагины также работают со специфичными полями. Треьим шагом мы создаём два экземпляра двух объектов, Первый экземпляр объекта Mikrotik Chateau, второй экземпляр объекта Slate AX1800. Эти два экзепляра готовы к моделированию. В качестве моделирования предлагаю соединить два маршрутизатора с помощью ethernet кабеля. Соответственно мы должны сделать класс data-channel, объект ethernet-cable и экземпляр
кабеля (длина как свойство экзепляра). По дороге мы должны стабилизировать систему плагинов, компиляции, валидации, генерации, отловить ошибки. Если разобрать моделирование взаимодействия двух маршрутизаторов, то на уровне классов, у маршрутизатора есть несколько каналов обмена данными. Два экземпляра класса маршрутизатор соединияются при помощи экземпляров класса data-channel. Соответственно у класса рутер должнен быть интерфейс для обмена по каналу данных data-channel. На уровне объектов ethernet кабель является объектом класса data-channel. У объекта Mikrotik есть список портов ethernet, и у объекта Slate есть список ethernet портов. Объект ethernet кабель исеет свойство подключаться в ethernet порт. Экземпляр объекта кабель с одного конца подключается в экземпляр объекта Mikrotik в конкретный порт, и с другого конца подключается в экземпляр объекта Slate AX1800 в ethernet порт с конкретными параметрами.
