 ADR 0104: Ansible Role Generation from Topology

  Статус: Proposed

  Ключевые решения
  ┌────────────────────┬────────────────────────────────────────────────────┐
  │       Аспект       │                      Решение                       │
  ├────────────────────┼────────────────────────────────────────────────────┤
  │ Новый плагин       │ AnsibleRoleGenerator (stage: generate, order: 235) │
  ├────────────────────┼────────────────────────────────────────────────────┤
  │ Генерируется       │ host_vars, group_vars, playbooks                   │
  ├────────────────────┼────────────────────────────────────────────────────┤
  │ Остаётся статичным │ Role tasks, handlers, templates                    │
  ├────────────────────┼────────────────────────────────────────────────────┤
  │ Триггер генерации  │ Capability markers (cap.network.vpn_gateway)       │
  ├────────────────────┼────────────────────────────────────────────────────┤
  │ Секреты            │ Только ссылки, SOPS lookup в runtime               │
  └────────────────────┴────────────────────────────────────────────────────┘
  Структура вывода

  generated/home-lab/ansible/
  ├── inventory/production/
  │   ├── hosts.yml              # С группами ролей
  │   ├── host_vars/*.yml        # Topology-derived
  │   └── group_vars/*.yml       # Role defaults
  └── playbooks/*.yml            # Role application

  Чек-лист реализации

  - Создать ansible_role_generator.py
  - Добавить манифест в plugins.yaml
  - Создать Jinja2 шаблоны
  - Добавить capability class
  - Интегрировать в deploy workflow
  - Добавить CI валидацию
  - Архивировать ручной inventory
