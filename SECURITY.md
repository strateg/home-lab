# Security Policy

## Supported Versions

Security fixes are provided for the active `main` branch.

## Reporting a Vulnerability

Please do not open public issues for security findings.

Preferred reporting channels:

1. GitHub Security Advisory (private report).
2. Direct contact with the author (repository owner) for security or permission-sensitive issues.

When reporting, include:

- affected files/paths and commit SHA,
- minimal reproduction steps,
- impact assessment,
- suggested mitigation (if available).

You can expect an acknowledgement within 3 business days.
