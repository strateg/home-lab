## ✅ ЧТО СДЕЛАНО

### 1. ADR 0064 Полностью Переработан

**Файл:** `adr/0064-os-taxonomy-object-property-model.md`

**Было:**
- Один класс `os` с подклассами
- Firmware как "вид OS"
- Путаница в терминологии

**Стало:**
- **Два независимых класса:**
  1. `class: firmware` (программная основа устройства)
  2. `class: os` (операционная система поверх firmware)
- Четкие capabilities от каждого класса
- Real-world примеры: PC, MacBook, MikroTik, PDU

### 2. Ключевые Концепции Зафиксированы

#### Firmware (низкий уровень)
- Всегда требуется для активных устройств
- Vendor/hardware-specific
- Примеры: BIOS, UEFI, RouterOS firmware, PDU firmware
- Capabilities: `cap.firmware.mikrotik`, `cap.firmware.uefi`, `cap.firmware.arch.x86_64`

#### OS (высокий уровень)
- Условно (может быть или нет)
- Embedded (часть firmware) или installable (независимый)
- Примеры: RouterOS 7, Debian 12, macOS 14, Windows 11
- Capabilities: `cap.os.linux`, `cap.os.debian.12`, `cap.os.init.systemd`

#### Связь
- OS не существует без firmware
- Firmware может существовать без OS (PDU)
- Device binds к firmware (required) + OS (conditional)

### 3. Семантика Именования

**Class → Object:**
- `class: firmware` → `object: mikrotik-routeros7-firmware`
- `class: os` → `object: routeros-7`

**Не:**
- ~~`class: os.firmware`~~ (старая модель)
- ~~`object: routeros-7-firmware`~~ (неправильно — это firmware, не OS)

### 4. Device Bindings

```yaml
# PC single-boot
bindings:
  firmware: obj.firmware.generic-uefi-x86
  os_primary: obj.os.debian-12-generic

# PC dual-boot (Windows + Linux)
bindings:
  firmware: obj.firmware.generic-uefi-x86
  os_primary: obj.os.windows-11
  os_secondary: obj.os.debian-12-generic

# PC triple-boot
bindings:
  firmware: obj.firmware.generic-uefi-x86
  os_primary: obj.os.windows-11
  os_secondary: obj.os.debian-12-generic
  os_tertiary: obj.os.freebsd-14

# MacBook (NO multi-boot)
bindings:
  firmware: obj.firmware.apple-silicon-m2
  os_primary: obj.os.macos-14
  # os_secondary: NOT ALLOWED

# MikroTik Router
bindings:
  firmware: obj.firmware.mikrotik-routeros7
  os_primary: obj.os.routeros-7

# PDU (no OS)
bindings:
  firmware: obj.firmware.apc-pdu-mgmt
  # os отсутствует (forbidden by policy)
```

### 5. Capabilities От Двух Слоев

```yaml
# Device capabilities = firmware + OS
effective_capabilities:
  from_firmware:
    - cap.firmware.mikrotik
    - cap.firmware.routeros
    - cap.firmware.arch.x86_64
  from_os:
    - cap.os.routeros
    - cap.os.routeros.7
    - cap.os.embedded
```

### 6. Device Policies

```yaml
# class: compute.pc (SUPPORTS multi-boot)
firmware_policy: required
os_policy: installable_allowed

bindings:
  firmware: {required: true}
  os_primary: {required: true}
  os_secondary: {required: false}  # dual-boot
  os_tertiary: {required: false}   # triple-boot

# class: compute.macbook (NO multi-boot)
firmware_policy: required
os_policy: embedded_only

bindings:
  firmware: {required: true}
  os_primary: {required: true}
  # os_secondary: NOT ALLOWED by vendor policy

# class: router.mikrotik
firmware_policy: required
os_policy: embedded_only

# class: power.pdu
firmware_policy: required
os_policy: forbidden
```

## 💡 КЛЮЧЕВЫЕ ВЫВОДЫ

### 1. Firmware ≠ OS
- **Firmware:** Программная основа устройства (низкий уровень)
- **OS:** Операционная система (высокий уровень)
- **Связь:** OS не работает без firmware

### 2. Два Независимых Класса
- `class: firmware` → hardware-bound
- `class: os` → runtime environment
- Device связывается с обоими

### 3. PDU — Firmware-Only Device
- **Не** "OS-less device" (старая терминология)
- **А** "firmware-only device" (правильно)
- Firmware есть (управление SNMP)
- OS нет (не нужна)

### 4. MikroTik — Firmware Contains Embedded OS
- Firmware: mikrotik-routeros7-firmware
- OS: routeros-7 (embedded)
- Две сущности, но жестко связаны

### 5. PC — Firmware + Installable OS
- Firmware: generic-uefi-x86-firmware
- OS: debian-12-generic (installable)
- Пользователь может менять OS
